SPRINT 3 (LOWER PRIORITY)

Y2 - Overdue Check Background Task
File: server/jobs/checkOverdueReservations.js or server/cron/overdue.js
Task/cron to find late returns and create penalties.

Details:
- Runs daily (8:00 AM) or on-demand
- Find all approved Reservations with endDate < today
- Check if Reservation.status === approved (not yet returned)
- Calculate daysLate = today - endDate
- Create Penalty doc with daysLate and penaltyAmount (daysLate * 5 or 10)
- Optional: Notify admin/student via email or flag

AI Prompt:
"Create server/jobs/checkOverdueReservations.js. On execution (daily), 
find all Reservation docs with status approved and endDate < today. For 
each, calculate daysLate. If no Penalty exists for that reservation, create 
one with penaltyAmount = daysLate * 5 (configurable penalty per day). Log 
results. Optional: use node-cron to schedule daily at 8:00 AM, or call 
manually from API endpoint. Export function or setup cron."

Commit: git commit -m "feat(backend): overdue penalty check job"

Details:

- Cannot be booked when status === Maintenance
- Check happens in checkAvailability() + reservation creation
- Technician can manually set status back to Available
